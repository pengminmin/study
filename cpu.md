## 冯诺依曼模型

1. 约定用二进制进行计算和存储
2. 定义计算机基本结构为 5 个部分
   1. 中央处理器 (CPU)
   2. 内存
   3. 输入设备
   4. 输出设备
   5. 总线

### 内存

1. 数据存储单位是一个 bit，即 0 或 1
2. 最小的存储单位是字节 byte，一个字节等于 8 位

### 中央处理器

1. 32 位 CPU 一次可以计算 4 个字节
2. 64 位 CPU 一次可以计算 8 个字节

CPU 内部还有一些组件，常见的有寄存器、控制单元和逻辑单元

#### 寄存器

1. 通用寄存器，用来存放需要进行运算的数据
2. 程序计数器，用来存储 CPU 要执行下一条指定的地址
3. 指定寄存器，用来存放指令

### 总线

1. 地址总线，用于指定 CPU 将要操作的内存地址
2. 数据总线，用于读写内存数据
3. 控制总线，用于发送和接收信号

## 线路位宽与 CPU 位宽

低电压表示 0，高电压表示 1

一位一位的传输方式称为串行，增加线路称为并行

32 位 CPU 最大只能操作 4GB 内存，就算装了 8GB 内存条也没用

64 位 CPU 理论寻址空间 2^64，实际 64 位系统寻址空间 2^48 = 256TB

## 总结

1. 64 位 CPU 可以一次计算超过 32 位的数字，而 32 位 CPU 如果要计算超过 32 位的数字，要份多步骤进行计算，效率没 64 位 CPU 高，所以只有运算大数字的时候，64 位 CPU 的优势才能提现处理，否则和 32 位 CPU 的计算性能相差不大。
2. 64 位 CPU 可以寻址更大的内存空间，32 位 CPU 最大的寻址空间是 4G，即使加了 8G 内存也只能用到 4G，64 位 CPU 理论最大寻址空间是 2^64，实际是 2^48 = 256TB
3. 32 位指定在 64 位机器上执行，需要一套兼容机制可以做到允许，但是 64 位指定在 32 位机器上执行就不行了，因为 32 位的寄存器存不下 64 位的指定
4. 硬件的 64 位和 32 位指的是 CPU 的位宽，软件的 64 位和 32 位指的是指令的位宽
